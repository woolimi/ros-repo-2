# Customer UI 명세

> **기술 스택:** Flask + SocketIO (포트 8501). 스마트폰 브라우저용 웹앱.
> **통신:** 채널 A (WebSocket, 브라우저 ↔ customer_web) + 채널 C (TCP, customer_web ↔ control_service)

---

## 구현 기능 목록

| # | 기능 | 연관 UR | 진입 조건 |
|---|---|---|---|
| 1 | 세션 확인 / 로그인 | UR-01, UR-02 | 앱 최초 진입 |
| 2 | 중복 사용 차단 | UR-05, UR-06 | 로봇이 이미 사용 중 |
| 3 | 주인 인형 등록 | UR-03 | 로그인 성공 후 (IDLE 상태) |
| 4 | 메인 쇼핑 화면 (TRACKING / TRACKING_CHECKOUT / WAITING) | UR-10, UR-12 | 등록 완료 후 |
| 5 | QR 상품 스캔 (장바구니 추가) | UR-30 | 메인 화면 → [담기] |
| 6 | 자연어 상품 검색 및 안내 | UR-32, UR-33 | 메인 화면 → [찾기] |
| 7 | 대기 모드 전환 | UR-12 | 메인 화면 → [대기하기] |
| 8 | 결제 팝업 | UR-40 | BoundaryMonitor 결제 구역 진입 (TRACKING 상태 유지) |
| 9 | 귀환 요청 (보내주기) | UR-42 | 메인/대기 화면 → [쇼핑 종료] |
| 10 | LOCKED 알림 | UR-60, UR-62 | 미결제 물건 있는 상태에서 귀환 시 |
| 11 | HALTED 알림 | UR-14, UR-61 | 배터리 부족으로 로봇 정지 시 |
| 12 | 쇼핑몰 맵 보기 | UR-20 | 상단 바 [🗺️ 맵] 버튼 (로그인 후 항상 접근 가능) |

## 화면 구성

### 1. `login.html` — 로그인

**역할:** 로봇 QR 스캔 또는 URL 진입 시 표시. 사용자 ID/PW 입력.

**UI 구성:**
```
┌─────────────────────┐
│   🛒 ShopPinkki     │
│                     │
│  아이디 [________]  │
│  비밀번호 [_______] │
│                     │
│    [ 로그인 ]       │
│                     │
│  ⚠ 오류 메시지 영역 │
└─────────────────────┘
```

**UI 요소:**
- 아이디 / 비밀번호 입력 필드
- [로그인] 버튼 — 클릭 시 채널 C `{"cmd": "login", ...}` 전송
- 오류 메시지 영역: "잘못된 아이디 또는 비밀번호", "이미 사용 중인 카트" 등

**플로우:**
```
진입 → session_check (채널 C)
    ├─ 세션 없음 → 로그인 폼 표시
    ├─ 세션 있음 (내 로봇) → main.html 리다이렉트
    └─ 로봇 사용 중 → blocked.html 리다이렉트

로그인 시도 → login (채널 C)
    ├─ 성공 → main.html 리다이렉트
    └─ 실패 → 오류 메시지 표시
```

---

### 2. `blocked.html` — 중복 사용 차단

**역할:** 이미 다른 사용자가 해당 로봇을 사용 중일 때 표시.

**UI 구성:**
```
┌─────────────────────┐
│   🚫 사용 불가      │
│                     │
│  현재 다른 사용자가 │
│  이 카트를 이용 중  │
│  입니다.            │
│                     │
│  [ 새로고침 ]       │
└─────────────────────┘
```

---

### 3. `main.html` — 메인 쇼핑 화면

**역할:** 로그인 후 전체 쇼핑 세션 동안 사용하는 단일 페이지. 로봇 상태(mode)에 따라 패널이 전환된다.

#### 3-A. 공통 상단 바

```
┌──────────────────────────────────────┐
│ 🤖 #54  TRACKING  🔋72%   [🗺️ 맵]  │
└──────────────────────────────────────┘
```
- 로봇 번호, 현재 모드 뱃지 (색상 구분), 배터리 %
- **[🗺️ 맵]** 버튼 — 클릭 시 맵 오버레이(3-I) 표시. 모든 mode에서 항상 활성화
- 1~2Hz WebSocket push로 자동 갱신

**모드 뱃지 색상:**

| 모드 | 뱃지 색 | 표시 텍스트 |
|---|---|---|
| `IDLE` | 파랑 | 등록 대기 |
| `TRACKING` | 초록 | 추종 중 |
| `TRACKING_CHECKOUT` | 청록 | 추종 중 (결제완료) |
| `GUIDING` | 노랑 | 안내 중 |
| `SEARCHING` | 주황 | 탐색 중 |
| `WAITING` | 하늘 | 대기 중 |
| `LOCKED` | 빨강 | 잠금 |
| `RETURNING` | 보라 | 귀환 중 |
| `HALTED` | 빨강 점멸 | 배터리 부족 |

---

#### 3-B. 인형 등록 패널 (mode=IDLE)

```
┌─────────────────────────────┐
│  📷 인형을 카메라에          │
│     보여주세요               │
│                             │
│       ⟳ 감지 중...          │
└─────────────────────────────┘
```
- 안내 문구 + 스피너 (IDLE 상태 진입 시 표시)
- 등록 완료 (`registration_done` WebSocket 수신) 시 → TRACKING 패널로 자동 전환

---

#### 3-C. 메인 쇼핑 패널 (mode=TRACKING / TRACKING_CHECKOUT / WAITING)

```
┌─────────────────────────────┐
│  [🔍 상품 찾기]  [📦 담기]  │
├─────────────────────────────┤
│  장바구니                    │
│  • 콜라       1,500원  [✕] │  ← 결제완료 항목은 ✓ 표시
│  • 감자칩     2,000원  [✕] │
│  ──────────────────────     │
│  합계         3,500원       │
├─────────────────────────────┤
│  [⏸ 대기하기]  [🚪 종료]  │
└─────────────────────────────┘
```

- **[상품 찾기]** — 검색 패널 열기 (3-D)
- **[담기]** — QR 스캔 패널 열기 (3-E)
- **장바구니** — WebSocket `cart` 수신마다 갱신. 결제 완료된 항목(`is_paid=True`)은 ✓ 표시 및 [✕] 비활성화. 미결제 항목만 [✕] 클릭으로 삭제 가능 → `{"cmd": "delete_item", "item_id": N}`
- **[대기하기]** — `{"cmd": "mode", "value": "WAITING"}` (TRACKING / TRACKING_CHECKOUT 상태에서 표시)
- **[따라가기]** — `{"cmd": "resume_tracking"}` (WAITING 상태에서 표시)
- **[쇼핑 종료]** — 미결제 물건 있으면 확인 다이얼로그 후 `{"cmd": "return"}` 전송

---

#### 3-D. 상품 검색 패널 (오버레이)

```
┌─────────────────────────────┐
│  🔍 상품 검색               │
│  [________________] [🎤]   │
│  [ 검색 ]  [ 닫기 ]         │
│                             │
│  → "음료 코너로 안내합니다" │
└─────────────────────────────┘
```
- 텍스트 입력 또는 STT (마이크 버튼, Web Speech API)
- [검색] → LLM 서버(채널 D) 질의 → zone_id 반환 → `{"cmd": "navigate_to", "zone_id": N}` 전송
- 결과 수신(`find_product_result`) → "X 코너로 안내합니다" 메시지 + GUIDING 이동 시작 (검색 패널 닫힘)
- 도착(`arrived`) 수신 → **도착 팝업(3-H)** 표시

---

#### 3-E. QR 스캔 패널 (오버레이)

```
┌─────────────────────────────┐
│  📷 상품 QR을 스캔하세요     │
│                             │
│  • 콜라  1,500원 ✓          │
│  • 감자칩 2,000원 ✓         │
│                             │
│  (30초 무활동 시 자동 종료)  │
│  [ 담기 완료 ]              │
└─────────────────────────────┘
```
- 스캔 성공마다 목록 갱신 (WebSocket `cart` 수신)
- [담기 완료] 또는 30초 타임아웃 → 패널 닫기, 추종 재개

---

#### 3-F. 결제 팝업 (결제 구역 진입 시, 모달 — mode=TRACKING 유지)

```
┌─────────────────────────────┐
│  💳 결제 구역 진입           │
│                             │
│  카트에 담긴 상품을          │
│  결제하시겠습니까?           │
│                             │
│  • 콜라       1,500원       │
│  • 감자칩     2,000원       │
│  ──────────────────────     │
│  합계         3,500원       │
│                             │
│  LCD의 QR을 스캔하여        │
│  결제를 진행해주세요.        │
│                             │
│  [ 계속 쇼핑하기 ]          │
└─────────────────────────────┘
```

- BoundaryMonitor 결제 구역 진입 시 자동 표시 (`checkout_zone_enter` WebSocket 수신)
- **SM 상태는 TRACKING 유지** — 결제 완료 전까지 출구 통과 불가
- 결제는 LCD에 표시된 QR 코드를 스캔하여 진행
- 결제 완료(`payment_done` WebSocket 수신) → 팝업 닫힘, 모드 뱃지 TRACKING_CHECKOUT으로 전환
- **[계속 쇼핑하기]** — 팝업 닫기, TRACKING 상태로 계속 쇼핑 (결제 구역 통과 불가 상태 유지)
- 미결제 상태에서 출구 방향 이동 시도 시 → "결제 후 통과 가능합니다" 알림 토스트

---

#### 3-G. LOCKED 알림 패널 (mode=LOCKED / RETURNING with is_locked_return)

```
┌─────────────────────────────┐
│  🔒 카트 잠금               │
│                             │
│  미결제 물건이 있어          │
│  카트가 잠겼습니다.          │
│                             │
│  카트가 충전소로 이동 중     │
│  입니다. 직원이 처리할       │
│  예정입니다.                 │
│                             │
│  잠시 기다려 주세요...       │
└─────────────────────────────┘
```
- `enter_locked` WebSocket 수신 시 전환
- 고객이 할 수 있는 조작 없음 (직원 처리 대기)
- 직원 처리 완료(`staff_resolved`) 수신 시 → 세션 종료 → login.html 리다이렉트

---

#### 3-H. 도착 팝업 (mode=GUIDING → WAITING 전환 시점, 모달)

```
┌─────────────────────────────┐
│                             │
│  📍 도착했습니다!            │
│                             │
│  [상품명] 코너에             │
│  도착했습니다.               │
│                             │
│        [ 확인 ]             │
│                             │
└─────────────────────────────┘
```
- `arrived` WebSocket 이벤트 수신 시 모달 팝업으로 표시
- 로봇은 WAITING 상태로 전환 (현재 위치에서 정지 대기)
- **[확인]** 클릭 → `{"cmd": "resume_tracking"}` 전송 → 이전 추종 상태로 복귀

---

#### 3-I. HALTED 알림 패널 (mode=HALTED)

```
┌─────────────────────────────┐
│  🔋 배터리 부족             │
│                             │
│  카트의 배터리가 부족하여    │
│  현재 위치에서 정지했습니다. │
│                             │
│  직원을 호출했습니다.        │
│  잠시 기다려 주세요.         │
└─────────────────────────────┘
```
- `enter_halted` WebSocket 수신 시 전환
- 고객이 할 수 있는 조작 없음 (직원 처리 대기)
- 직원 처리 완료(`staff_resolved`) 수신 시 → 세션 종료 → login.html 리다이렉트

---

#### 3-J. 맵 오버레이 (모든 mode에서 접근 가능, 전체화면 모달)

```
┌─────────────────────────────┐
│  🗺️ 마트 지도        [ ✕ ] │
├─────────────────────────────┤
│                             │
│   ┌──────────────────────┐  │
│   │  [맵 이미지]         │  │
│   │   ● 내 로봇 #54      │  │  ← 진한 마커 (강조)
│   │   ○ 로봇 #18         │  │  ← 연한 마커 (투명도 낮춤)
│   └──────────────────────┘  │
│                             │
│  ● 내 로봇  ○ 다른 로봇     │
└─────────────────────────────┘
```

**UI 요소:**
- 맵 영역: `shop.pgm` 기반 Canvas 이미지 위에 로봇 위치 마커 오버레이
  - **내 로봇 (●, 파란색, 불투명)**: 강조 표시
  - **다른 로봇 (○, 회색, 투명도 40%)**: 연하게 표시하여 시각적 구분
- 하단 범례: 파란색 = 내 로봇, 회색 = 다른 로봇
- 핀치 줌 지원 (모바일 스마트폰 대응)

**좌표 변환 (Canvas 픽셀 기준):**
```
px_x = (pos_x - origin_x) / resolution
px_y = img_height_px - (pos_y - origin_y) / resolution   # Y축 반전
```
> `resolution`, `origin`은 `shoppinkki_nav/maps/shop.yaml`에서 읽어 초기화.

## 유저 플로우

```
[스마트폰 QR 스캔 / URL 접속]
        ↓
  session_check
  ├─ 로봇 사용 중 → blocked.html (새로고침 대기)
  └─ 비어있음 → login.html
        ↓
  ID / PW 입력 → 로그인
  ├─ 실패 → 오류 메시지 재입력
  └─ 성공 → main.html
        ↓
  ┌─ [인형 등록 패널: IDLE] ───────────────────────┐
  │  인형을 카메라에 보여줌 → YOLO 감지 → 등록 완료 │
  └───────────────────────────────────────────────┘
        ↓
  ┌─ [메인 쇼핑 패널: TRACKING] ───────────────────────────────────────────────┐
  │                                                                            │
  │  [🗺️ 맵] (상단 바, 항상 접근 가능)                                        │
  │      → 내 로봇 진한 마커, 다른 로봇 연한 마커로 실시간 표시                │
  │      └─ [✕] → 맵 닫기 (현재 mode 유지)                                   │
  │                                                                            │
  │  [담기] → QR 스캔 패널 → 상품 스캔 → [담기 완료] → TRACKING 복귀         │
  │                                                                            │
  │  [찾기] → 텍스트/STT 검색 → 안내 시작 (GUIDING)                           │
  │      └─ 도착 팝업(WAITING) → [확인] → 이전 추종 상태로 복귀               │
  │                                                                            │
  │  [대기하기] → WAITING 패널                                                │
  │      ├─ [따라가기] → 이전 추종 상태로 복귀                                   │
  │      └─ 타임아웃 → 미결제 물건 있으면 LOCKED, 없으면 RETURNING             │
  │                                                                            │
  │  결제 구역 진입 (자동, TRACKING 상태 유지)                                 │
  │      → 결제 팝업 표시 + LCD 결제 QR                                        │
  │      ├─ LCD QR 결제 완료 → TRACKING_CHECKOUT 전환                          │
  │      │     └─ 출구 통과 가능. 결제구역 재진입 시 TRACKING으로 복귀          │
  │      └─ [계속 쇼핑] → 팝업 닫기 (출구 통과 여전히 불가)                   │
  │                                                                            │
  │  [쇼핑 종료] → 미결제 물건 확인 다이얼로그                                 │
  │      ├─ 물건 없음 → RETURNING → 충전소 도착 → 세션 종료 → login.html      │
  │      └─ 물건 있음 → LOCKED → 충전소 귀환 중 알림 → 직원 처리 → 세션 종료  │
  │                                                                            │
  │  배터리 부족 (자동) → HALTED → 정지 알림 → 직원 처리 → 세션 종료           │
  │                                                                            │
  └───────────────────────────────────────────────────────────────────────── ┘
```

## WebSocket 이벤트 처리 요약

| 수신 이벤트 | UI 동작 |
|---|---|
| `status` (mode 변경) | 모드 뱃지 갱신. mode에 따라 패널 전환. 맵 오버레이 마커 위치 갱신 |
| `cart` | 장바구니 목록 갱신. `is_paid=True` 항목 ✓ 표시 |
| `registration_done` | 인형 등록 패널 → 메인 쇼핑 패널 전환 |
| `checkout_zone_enter` | 결제 팝업(3-F) 표시 (SM 상태 변경 없음) |
| `payment_done` | 결제 팝업 닫힘, 모드 뱃지 TRACKING_CHECKOUT 전환 |
| `checkout_blocked` | "결제 후 통과 가능합니다" 토스트 |
| `find_product_result` | "X 코너로 안내합니다" 표시 |
