# SC-41: 장바구니 항목 삭제

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` 내부 (SM 변경 없음) |
| **트리거** | 장바구니 [✕] 클릭 → `delete_item` |
| **우선순위** | MEDIUM |
| **연관** | UR-31, SR-62 |

---

## 사전 조건

- SM = `TRACKING`
- 장바구니에 미결제(`is_paid=0`) 항목 2개 이상 존재

## 시험 절차

1. **[앱] 메인 쇼핑 패널 장바구니 목록 확인**
   - 미결제 항목: [✕] 버튼 활성화
   - 결제 완료 항목(`is_paid=True`): ✓ 표시, [✕] 비활성화

2. **미결제 항목 옆 [✕] 클릭**
   - [앱] → customer_web: `{"cmd": "delete_item", "item_id": N}`
   - control_service → Pi: `{"cmd": "delete_item", "item_id": N}`
   - Pi: `delete_cart_item(N)` → DB 삭제 → `/robot_54/cart` 재발행

3. **[앱] 장바구니 갱신 확인**
   - 삭제 항목 제거, 합계 갱신

4. **결제 완료 항목 [✕] 클릭 시도**
   - 예상: 버튼 비활성화 → 클릭 불가

## 예상 결과

- CART_ITEM DB에서 해당 `item_id` 삭제 확인
- [앱] 실시간 장바구니 목록 갱신
- 결제 완료 항목은 삭제 불가 (UI 차단)
- SM 변경 없음
