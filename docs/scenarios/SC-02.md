# SC-02: 세션 재접속 (쿠키 유효)

| 항목 | 내용 |
|---|---|
| **상태 전환** | 세션 유효 → 로그인 건너뜀 → 메인화면 직행 |
| **트리거** | `session_check` 응답 = 활성 세션 있음 |
| **우선순위** | MEDIUM |
| **연관** | UR-04, SR-11 |

---

## 사전 조건

- SC-01 완료 상태 (SM = `TRACKING`, 세션 활성)
- 이전 접속에서 발급된 쿠키가 브라우저에 유지 중

## 시험 절차

1. **브라우저에서 QR URL 재접속** (`http://localhost:8501/?robot_id=54`)

2. **customer_web `session_check` 요청 확인**
   - control_service 로그에서 `session_check` 수신 및 활성 세션 응답 확인

3. **리다이렉트 경로 확인**

## 예상 결과

- `login.html` 표시 없음
- 인형 등록 패널(3-B) 표시 없음
- 메인 쇼핑 패널(3-C) 직접 표시
- 장바구니 목록 이전 상태 유지
- [앱] 상단 뱃지 = 현재 SM 상태 반영

## 실패 케이스

- 세션 만료(`expires_at < NOW()`) 후 접속 시 → `login.html` 으로 이동해야 함
- `is_active = 0` 세션으로 접속 시 → `login.html` 으로 이동해야 함
