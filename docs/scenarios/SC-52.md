# SC-52: 결제 완료 후 쇼핑 재개

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING_CHECKOUT` → `TRACKING` |
| **트리거** | BoundaryMonitor `on_checkout_reenter` → `enter_tracking` |
| **우선순위** | MEDIUM |
| **연관** | UR-40b, SR-32b |

---

## 사전 조건

- SM = `TRACKING_CHECKOUT` (SC-50 완료 후)
- 로봇이 결제 구역 출구 방향으로 이동 후 다시 결제 구역 안쪽으로 복귀

## 시험 절차

1. **SM = `TRACKING_CHECKOUT` 확인**

2. **로봇을 결제 구역 안쪽(출구 반대 방향)으로 이동**
   - 인형을 결제 구역 안쪽으로 유도
   - BoundaryMonitor: `on_checkout_reenter` 콜백 호출

3. **SM = `TRACKING` 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: TRACKING
   ```

4. **장바구니 상태 확인**
   - 이전 결제 완료 항목: `is_paid=1` 유지 (✓ 표시)
   - 새로 추가하는 항목: `is_paid=0`

5. **QR 스캔으로 새 항목 추가** (SC-40 참조)
   - 새 항목 `is_paid=0` 으로 추가됨

6. **다시 결제 구역 진입 시**
   - 결제 팝업에 `is_paid=0` 항목만 표시 확인

## 예상 결과

- SM = `TRACKING`
- [앱] 뱃지: "추종 중 (결제완료)" → "추종 중"
- 기결제 항목 `is_paid=1` 유지, 새 항목 `is_paid=0` 추가 가능
- 다음 결제 구역 진입 시 미결제 항목만 결제 대상
