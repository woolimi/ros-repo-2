# SC-70: 배터리 부족 → 즉시 정지 (HALTED)

| 항목 | 내용 |
|---|---|
| **상태 전환** | `*` (임의 상태) → `HALTED` |
| **트리거** | `enter_halted` (`source='*'`, 배터리 임계값 이하) |
| **우선순위** | HIGH |
| **연관** | UR-14, SR-31, SR-46, SR-96 |

---

## 사전 조건

- SM = 임의 활성 상태 (TRACKING, GUIDING, RETURNING 등 — 각각 테스트)
- `BATTERY_THRESHOLD = 20`% (테스트 시 단축: `config.py`에서 90%로 임시 상향)

## 시험 절차

### 방법 A — 실 배터리 소모 (실 환경)

1. 배터리 잔량이 임계값 이하로 떨어질 때까지 대기

### 방법 B — 강제 트리거 (개발/테스트)

1. **SM이 임의 활성 상태인지 확인** (예: SM = `TRACKING`)

2. **배터리 부족 강제 트리거**
   ```bash
   # main_node.py에 테스트 엔드포인트 또는 직접 토픽 발행
   ros2 topic pub /robot_54/cmd std_msgs/String \
     '{"data": "{\"cmd\": \"test_battery_low\"}"}'
   ```
   또는 `config.py`의 `BATTERY_THRESHOLD = 90`으로 임시 설정 후 정상 배터리 확인 시 트리거

3. **SM = `HALTED` 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: HALTED
   ```

4. **로봇 즉시 정지 확인**
   ```bash
   ros2 topic echo /robot_54/cmd_vel
   # linear.x = 0, angular.z = 0
   ```
   - 진행 중인 Nav2 Goal 취소 확인

5. **[앱] HALTED 알림 패널(3-I) 표시 확인**

6. **[관제] 스태프 호출 패널 HALTED 항목 표시 확인**

7. **LED 확인** — 흰색 점멸

8. **LCD 확인** — "배터리 부족 — 직원을 호출합니다"

## 예상 결과

- SM = `HALTED` (어떤 상태에서 진입해도 동일)
- 로봇 즉시 정지 (현재 위치 그대로)
- 자동 전환 없음
- [앱] HALTED 알림 패널 표시 (조작 불가)
- [관제] 스태프 호출 패널에 `HALTED` 항목 추가
- LED = 흰색 점멸
- `STAFF_CALL_LOG`에 `event_type='HALTED'` 기록
