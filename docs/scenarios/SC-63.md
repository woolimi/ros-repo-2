# SC-63: LOCKED 도착 → staff_resolved → 세션 종료

| 항목 | 내용 |
|---|---|
| **상태 전환** | `CHARGING`(is_locked_return=True) → `CHARGING`(is_locked_return=False) |
| **트리거** | `staff_resolved` 명령 |
| **우선순위** | HIGH |
| **연관** | UR-62, UR-53, SR-51, SR-57, SR-59 |

---

## 사전 조건

- SC-62 완료 후 상태: SM = `CHARGING`, `is_locked_return = True`
- LED = 빨간색 점멸
- [관제] 스태프 호출 패널에 LOCKED 항목 표시 중

## 시험 절차

1. **[관제] 스태프 호출 패널에서 Robot#54 LOCKED 항목 확인**

2. **직원이 물리적으로 충전소에 도착하여 카트 확인** (실 환경)

3. **[관제] [잠금 해제] 버튼 클릭**
   - 확인 다이얼로그 → 확인
   - admin_ui → control_service: `{"cmd": "staff_resolved", "robot_id": 54}`
   - control_service → Pi: `{"cmd": "staff_resolved"}`

4. **Pi 처리 확인**
   - `is_locked_return = False`
   - `terminate_session()` 호출: `SESSION.is_active=0`, `CART_ITEM` 삭제
   - `ROBOT.active_user_id = NULL`
   - `staff_call_log.resolved_at` 갱신

5. **LED 변화 확인** — 빨간색 점멸 → 빨강(일반 충전 중)

6. **[앱] 확인** (고객 측)
   ```
   staff_resolved WebSocket 이벤트 → login.html 리다이렉트
   ```

7. **[관제] 스태프 호출 패널 항목 "✓ 처리됨" 회색 처리 확인**

## 예상 결과

- `is_locked_return` = False
- SM = `CHARGING` 유지 (상태 전환 없음, 플래그만 해제)
- LED = 빨강 (일반 충전 신호)
- `SESSION.is_active` = 0, `ROBOT.active_user_id` = NULL
- `STAFF_CALL_LOG.resolved_at` 갱신
- EVENT_LOG에 `STAFF_RESOLVED` 기록
- [앱] `login.html` 리다이렉트
