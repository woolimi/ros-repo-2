# SC-11: 주인 놓침 → 탐색 타임아웃 → WAITING

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → `SEARCHING` → `WAITING` |
| **트리거** | `enter_searching` → `enter_waiting` (탐색 타임아웃) |
| **우선순위** | HIGH |
| **연관** | UR-11, UR-12, SR-42 |

---

## 사전 조건

- SM = `TRACKING`
- `SEARCH_TIMEOUT = 30.0`초 (테스트 시 단축 가능: `config.py`에서 임시 조정)

## 시험 절차

1. **인형을 카메라 시야에서 완전히 제거** (인형 없는 상태 유지)
   - BT 1: `N_MISS_FRAMES` 도달 → `enter_searching`

2. **SM = `SEARCHING` 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```

3. **탐색 타임아웃 대기** (`SEARCH_TIMEOUT`초)
   - BT 2: 30초 경과 + 인형 미감지 → `enter_waiting`

4. **SM 전환 확인**

5. **로봇 정지 확인** (`/cmd_vel` = zero)

## 예상 결과

- SM = `WAITING`
- [앱] 뱃지: "탐색 중" → "대기 중"
- LCD: "주인을 찾고 있습니다" → "대기 중"
- LED: 주황 → 파랑
- 로봇 정지 (회전 종료)
- `previous_tracking_state` = `TRACKING` (유지 — WAITING 진입 시 저장)

## 후속 확인

- 이 상태에서 SC-20(앱 따라가기) 또는 SC-21/SC-22(타임아웃) 시나리오로 연계 가능
