# SC-51: 미결제 상태 출구 통과 차단

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` 유지 (SM 변경 없음) |
| **트리거** | BoundaryMonitor `on_checkout_exit_blocked` → 로봇 정지 |
| **우선순위** | HIGH |
| **연관** | UR-40a, SR-32a |

---

## 사전 조건

- SM = `TRACKING` (결제 미완료 상태)
- 장바구니에 미결제 항목 있음
- 로봇이 결제 구역(zone 150) 안에 위치

## 시험 절차

1. **SM = `TRACKING`, 결제 팝업(3-F) 표시 중 상태 확인** (SC-50 절차 1~3 완료)

2. **결제 팝업의 [계속 쇼핑하기] 클릭** — 팝업 닫기, SM = `TRACKING` 유지

3. **로봇을 출구 방향(결제 구역 경계 초과)으로 이동 시도**
   - 인형을 출구 방향으로 유도
   - 또는 Gazebo에서 AMCL pose를 출구 경계 좌표로 시뮬레이션

4. **BoundaryMonitor `on_checkout_exit_blocked` 콜백 확인**

5. **로봇 정지 확인**
   ```bash
   ros2 topic echo /robot_54/cmd_vel
   # linear.x = 0, angular.z = 0
   ```

6. **[앱] 알림 확인**

## 예상 결과

- 로봇 출구 방향 이동 차단 (즉시 정지)
- SM = `TRACKING` 유지 (상태 변경 없음)
- [앱] "결제 후 통과 가능합니다" 토스트 (`checkout_blocked` 이벤트)
- [앱] 결제 팝업 재표시 (또는 토스트만)

## 해제 조건

- 결제 완료(SC-50 절차 4) 후 SM = `TRACKING_CHECKOUT` 전환 시 출구 통과 허용됨
