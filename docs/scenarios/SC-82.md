|---|---|

## 사전 조건

- 로봇 #54가 정상 동작 중 (status 1~2Hz 발행)
- control_service cleanup 스레드 동작 중 (10초 주기)
- `ROBOT_TIMEOUT_SEC = 30`초

## 시험 절차

1. **로봇 정상 동작 확인** — [관제] Robot#54 카드 정상 표시

2. **로봇 네트워크 연결 차단** (또는 ROS 노드 강제 종료)
   ```bash
   # Pi에서 ROS 노드 종료 또는 네트워크 차단
   # 또는 테스트: main_node 프로세스 kill
   ```

3. **30초 대기** — cleanup 스레드가 `last_seen` 초과 감지

4. **control_service 처리 확인**
   - `ROBOT.current_mode = 'OFFLINE'`
   - `ROBOT.active_user_id = NULL`
   - admin_ui에 `{"type": "offline", "robot_id": 54}` push

5. **[관제] 확인**
   - Robot#54 카드 전체 회색 처리
   - 모드 뱃지 = "오프라인"
   - 맵 오버레이 아이콘 × 표시, 마지막 위치 유지

6. **로봇 네트워크 복구** → status 발행 재개

7. **온라인 복귀 확인**
   - `{"type": "online", "robot_id": 54}` push
   - [관제] 카드 정상 복구
