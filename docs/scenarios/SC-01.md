# SC-01: 로그인 → 인형 등록 → 추종 시작

| 항목 | 내용 |
|---|---|
| **상태 전환** | `CHARGING` → `IDLE` → `TRACKING` |
| **트리거** | `charging_completed` → `enter_tracking` |
| **우선순위** | HIGH |
| **연관** | UR-01, UR-02, UR-03, SR-10, SR-23 |

---

## 사전 조건

- 로봇 전원 ON, SM = `CHARGING`
- control_service, customer_web 기동
- DB에 테스트 계정 존재 (`user_id: test01`, `password: 1234`)
- 인형 소품 준비

## 시험 절차

1. **충전 완료 트리거 발송**
   ```bash
   ros2 topic pub /robot_54/cmd std_msgs/String \
     '{"data": "{\"cmd\": \"charging_completed\"}"}'
   ```
   - 예상: SM = `IDLE`, LCD = QR 코드 + "인형을 카메라에 보여주세요" 안내

2. **[앱] QR 스캔 또는 `http://localhost:8501/?robot_id=54` 접속**
   - 예상: `login.html` 표시

3. **[앱] ID `test01` / PW `1234` 입력 → [로그인] 클릭**
   - 예상: `start_session` cmd → Pi 수신, 메인화면 이동, 인형 등록 패널(3-B) 표시, LCD "인형을 카메라에 보여주세요"

4. **인형을 Pi 카메라 앞에 제시 (3~5초 유지)**
   - 예상: YOLO 감지 → ReID 템플릿 등록 완료

5. **등록 완료 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```

## 예상 결과

- `ROBOT.current_mode` = `TRACKING`
- `SESSION.is_active` = 1, `ROBOT.active_user_id` = `test01`
- [앱] 인형 등록 패널 → 메인 쇼핑 패널(3-C) 자동 전환 (`registration_done` 이벤트 수신)
- [앱] 상단 뱃지 = 초록 "추종 중"
- LCD = "추종 중" 상태 메시지
- LED = 초록
