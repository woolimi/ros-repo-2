# SC-81: admin_goto — 위치 호출

| 항목 | 내용 |
|---|---|
| **상태 전환** | `IDLE` → Nav2 이동 (SM 변경 없음) |
| **트리거** | `admin_goto` |
| **우선순위** | LOW |
| **연관** | UR-50, SR-94 |

---

## 사전 조건

- SM = `IDLE` (로봇 유휴 상태)
- Nav2 기동, 초기 위치 설정 완료

## 시험 절차

1. **[관제] Robot#54 카드 모드 = `IDLE` 확인**

2. **[관제] 맵 오버레이 클릭** → 목표 마커(파란 십자+원) 표시

3. **[관제] [이동 명령] 버튼 클릭**
   - admin_ui → control_service: `{"cmd": "admin_goto", "robot_id": 54, "x": ..., "y": ..., "theta": 0.0}`
   - control_service → Pi: `{"cmd": "admin_goto", "x": ..., "y": ..., "theta": 0.0}`
   - Pi: Nav2 Goal 전송

4. **로봇 이동 확인** (맵 오버레이 마커 이동)

5. **IDLE 상태가 아닌 로봇에 admin_goto 시도 (거부 확인)**
   - SM = `TRACKING` 상태 로봇에 이동 명령
   - 예상: control_service 거부 → `admin_goto_rejected` push

## 예상 결과 (정상)

- 로봇이 지정 좌표로 이동
- SM = `IDLE` 유지 (상태 전환 없음)
- 도착 후 목표 마커 제거

## 예상 결과 (거부)

- SM ≠ IDLE → [관제] 오류 토스트: "해당 로봇은 IDLE 상태가 아닙니다"
- 로봇 동작 변경 없음
