# SC-20: 사용자 [대기하기] → [따라가기] 복귀

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → `WAITING` → `TRACKING` |
| **트리거** | `enter_waiting` (앱) → `resume_tracking` (앱) |
| **우선순위** | MEDIUM |
| **연관** | UR-12, SR-34 |

---

## 사전 조건

- SM = `TRACKING`

## 시험 절차

1. **[앱] 메인 쇼핑 패널(3-C)에서 [⏸ 대기하기] 클릭**
   - customer_web → control_service: `{"cmd": "mode", "value": "WAITING"}`
   - control_service → Pi: `{"cmd": "mode", "value": "WAITING"}`

2. **SM 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```
   - 예상: `mode = WAITING`

3. **[앱] UI 확인**
   - [⏸ 대기하기] 버튼이 [▶ 따라가기] 버튼으로 전환
   - 뱃지: "대기 중"

4. **[앱] [▶ 따라가기] 클릭**
   - customer_web → control_service: `{"cmd": "resume_tracking"}`
   - control_service → Pi: `{"cmd": "resume_tracking"}`
   - Pi: `sm.resume_tracking()` → `previous_tracking_state` 기반 전환

5. **SM 전환 확인**

## 예상 결과

- SM = `TRACKING` (이전 상태가 TRACKING)
- [앱] 뱃지: "대기 중" → "추종 중"
- LED: 파랑 → 초록
- BT 1 재시작 (추종 루프 재개)

## 변형 테스트

- `previous_tracking_state = TRACKING_CHECKOUT` 상태에서 동일 절차
  → 복귀 후 SM = `TRACKING_CHECKOUT`, 뱃지 = "추종 중 (결제완료)"
