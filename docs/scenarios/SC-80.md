# SC-80: 관제 강제 종료

| 항목 | 내용 |
|---|---|
| **상태 전환** | `*` (임의 활성 상태) → `CHARGING` |
| **트리거** | `force_terminate` |
| **우선순위** | MEDIUM |
| **연관** | UR-52, SR-36, SR-94 |

---

## 사전 조건

- SM = 임의 활성 상태 (TRACKING, GUIDING, WAITING 등)
- 강제 종료 버튼 활성 조건: CHARGING·OFFLINE·HALTED·LOCKED 제외

## 시험 절차

1. **[관제] Robot#54 카드에서 현재 모드 확인** (예: `TRACKING`)

2. **[관제] [강제 종료] 버튼 클릭**
   - 확인 다이얼로그 표시 → 확인
   - admin_ui → control_service: `{"cmd": "force_terminate", "robot_id": 54}`
   - control_service → Pi: `{"cmd": "force_terminate"}`

3. **Pi 처리 확인**
   - 진행 중인 BT 강제 중단
   - Nav2 Goal 취소
   - `terminate_session()` 호출
   - SM = `CHARGING`

4. **SM 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: CHARGING
   ```

5. **[관제] 카드 뱃지 CHARGING 전환 확인**

6. **[앱] 세션 종료 → login.html 리다이렉트 확인**

## 예상 결과

- SM = `CHARGING`
- 세션 종료: `SESSION.is_active=0`
- `ROBOT.active_user_id` = NULL
- EVENT_LOG에 `FORCE_TERMINATE` 기록
- LED = 빨강

## 비활성화 케이스 확인

| 상태 | 버튼 상태 |
|---|---|
| `CHARGING` | 비활성화 |
| `OFFLINE` | 비활성화 |
| `HALTED` | 비활성화 (staff_resolved 사용) |
| `LOCKED` | 비활성화 (staff_resolved 사용) |
| `TRACKING` | **활성화** |
| `RETURNING` | **활성화** |
