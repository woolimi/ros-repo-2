# SC-61: 정상 귀환 — TRACKING_CHECKOUT 상태

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING_CHECKOUT` → `RETURNING` → `CHARGING` |
| **트리거** | `enter_returning` (보내주기, 모든 항목 결제 완료) → `enter_charging` |
| **우선순위** | HIGH |
| **연관** | UR-42, UR-43, SR-33 |

---

## 사전 조건

- SM = `TRACKING_CHECKOUT`
- 장바구니 전체 항목 `is_paid=1` (미결제 없음)

## 시험 절차

1. **[앱] [🚪 쇼핑 종료] 클릭**
   - `has_unpaid_items()` = False → `enter_returning`

2. **SM = `RETURNING` 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: RETURNING, is_locked_return: false
   ```

3. **BT 5 동작 + 충전소 도착** (SC-60 절차 3~4와 동일)

4. **SM = `CHARGING` 전환 확인**

## 예상 결과

- SM = `CHARGING`, `is_locked_return` = False (일반 귀환)
- 세션 종료, 장바구니 삭제
- [앱] `login.html` 리다이렉트
- LED: 초록 → 보라 → 빨강 (잠금 신호 없음)

## SC-60과의 차이

- 시작 상태가 `TRACKING_CHECKOUT`
- `is_locked_return` = False 확인이 핵심 (LOCKED 귀환과 구분)
