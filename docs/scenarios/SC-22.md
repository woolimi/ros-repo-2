# SC-22: WAITING 타임아웃 → 미결제 LOCKED 귀환

| 항목 | 내용 |
|---|---|
| **상태 전환** | `WAITING` → `LOCKED` → `RETURNING` → `CHARGING` |
| **트리거** | `enter_locked` (타임아웃 + 미결제) → `enter_returning` (자동) → `enter_charging` |
| **우선순위** | MEDIUM |
| **연관** | UR-12, UR-42, UR-60, SR-34, SR-45, SR-55 |

---

## 사전 조건

- SM = `WAITING`
- 장바구니에 미결제 물건 있음 (`is_paid=0` 항목 존재)
- `WAITING_TIMEOUT` 임시 단축 설정

## 시험 절차

1. **SM = `WAITING`, 미결제 항목 존재 확인**
   ```bash
   ros2 topic echo /robot_54/cart
   # is_paid=false 항목 확인
   ```

2. **WAITING_TIMEOUT 대기**
   - SM: `has_unpaid_items()` = True → `enter_locked`

3. **SM = `LOCKED` 전환 확인**
   - `is_locked_return` = True 설정 확인
   - LED = 빨간색 점멸 확인

4. **`on_enter_LOCKED` 콜백: 즉시 `enter_returning` 자동 전환 확인**
   - SM = `LOCKED` 체류 시간 매우 짧음 (자동 귀환 시작)
   - SM = `RETURNING` 진입 후 `is_locked_return` 플래그 유지 확인

5. **BT 5 동작 + 충전소 도착**
   - LED = 빨간색 점멸 유지 (RETURNING 중에도)

6. **SM = `CHARGING` 전환 후 `is_locked_return` 여전히 True 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # is_locked_return: true
   ```

7. **[관제] 스태프 호출 패널에 LOCKED 항목 표시 확인**

## 예상 결과

- SM = `CHARGING`, `is_locked_return` = True
- LED = 빨간색 점멸 (CHARGING 상태임에도 유지)
- [관제] 스태프 호출 패널: Robot#54 LOCKED 항목 표시 (미처리)
- [앱] LOCKED 알림 패널(3-G) 표시 중
- 세션 아직 활성 (staff_resolved 전까지 유지)

## 후속

- SC-63으로 연계하여 `staff_resolved` 처리 검증
