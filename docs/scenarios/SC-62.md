# SC-62: 미결제 보내주기 → LOCKED 귀환

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → `LOCKED` → `RETURNING` → `CHARGING` |
| **트리거** | `enter_locked` (보내주기 + 미결제) → 자동 `enter_returning` → `enter_charging` |
| **우선순위** | HIGH |
| **연관** | UR-42, UR-60, SR-33, SR-45, SR-55, SR-56 |

---

## 사전 조건

- SM = `TRACKING`
- 장바구니에 미결제 항목(`is_paid=0`) 존재

## 시험 절차

1. **[앱] [🚪 쇼핑 종료] 클릭**
   - `has_unpaid_items()` = True → 확인 다이얼로그 표시
   - "미결제 물건이 있습니다. 그래도 보내시겠습니까?" 확인

2. **[앱] 확인 클릭**
   - `{"cmd": "return"}` → Pi: `enter_locked`

3. **`on_enter_LOCKED` 콜백 확인**
   - `is_locked_return = True` 설정
   - 즉시 `enter_returning` 자동 전환

4. **SM = `RETURNING`, `is_locked_return = True` 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: RETURNING, is_locked_return: true
   ```

5. **LED 확인** — 빨간색 점멸 (RETURNING 상태임에도 잠금 신호)

6. **[앱] LOCKED 알림 패널(3-G) 표시 확인**
   - "카트가 충전소로 이동 중입니다. 직원이 처리할 예정입니다."

7. **BT 5 동작 + 충전소 도착**
   - `enter_charging` → SM = `CHARGING`

8. **SM = `CHARGING`, `is_locked_return = True` 여전히 유지 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: CHARGING, is_locked_return: true
   ```

9. **LED 확인** — 빨간색 점멸 유지 (CHARGING 상태임에도)

10. **[관제] 스태프 호출 패널 LOCKED 항목 표시 확인**

## 예상 결과

- SM = `CHARGING`, `is_locked_return` = True
- LED = 빨간색 점멸 (상태와 무관, 플래그 우선)
- 세션 아직 활성 (`is_active=1`)
- [관제] 스태프 호출 패널: `LOCKED` 미처리 항목

## 후속

- SC-63으로 이어서 `staff_resolved` 처리
