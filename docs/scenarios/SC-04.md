# SC-04: 중복 로그인 차단

| 항목 | 내용 |
|---|---|
| **상태 전환** | — (SM 변경 없음) |
| **트리거** | `login` 요청 시 해당 user_id가 이미 다른 로봇에 활성 세션 존재 |
| **우선순위** | LOW |
| **연관** | UR-06, SR-12 |

---

## 사전 조건

- `test01`이 로봇 #54에서 이미 활성 세션으로 사용 중
- 다른 기기에서 로봇 #18 QR URL로 접속하여 `test01` 로그인 시도

## 시험 절차

1. **로봇 #18 URL 접속** (`http://localhost:8501/?robot_id=18`) → `login.html` 표시

2. **ID `test01` / PW `1234` 입력 → [로그인] 클릭**

3. **control_service 처리 확인**
   - `SESSION` 테이블: `test01`의 `is_active=1` 세션이 다른 `robot_id`에 존재

4. **화면 확인**

## 예상 결과

- 로그인 거부
- `login.html` 오류 메시지: "이미 다른 카트를 사용 중입니다."
- 로봇 #18 SM 상태 변경 없음
- 로봇 #54 세션 영향 없음
