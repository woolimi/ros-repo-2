# SC-31: 상품 안내 중 Nav2 실패 → 추종 복귀

| 항목 | 내용 |
|---|---|
| **상태 전환** | `GUIDING` → `TRACKING` |
| **트리거** | Nav2 실패 → `sm.resume_tracking()` |
| **우선순위** | MEDIUM |
| **연관** | UR-33, SR-44 |

---

## 사전 조건

- SM = `GUIDING` (SC-30 절차 1~4 완료 후 이동 중)
- Nav2 실패를 유발할 수 있는 환경 준비

## 시험 절차

1. **SM = `GUIDING`, 로봇 이동 중 확인**

2. **Nav2 실패 유발**
   - Gazebo: 목적지 전방에 통과 불가 장애물 배치
   - 실 환경: 맵 미등록 구역으로 Goal 설정 (또는 Nav2 강제 취소)
   ```bash
   ros2 action send_goal /navigate_to_pose nav2_msgs/action/NavigateToPose \
     "{pose: {header: {frame_id: 'map'}, pose: {position: {x: 99.0, y: 99.0}}}}"
   ```

3. **BT 4 실패 처리 확인**
   - BT 4: Nav2 FAILURE → `sm.resume_tracking()` 호출

4. **SM 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```

5. **[앱] 알림 확인**

## 예상 결과

- SM = `TRACKING` (이전 상태가 TRACKING이었으므로)
- [앱] "안내에 실패했습니다" 토스트 표시 (`nav_failed` 이벤트)
- [앱] 뱃지: "안내 중" → "추종 중"
- LED: 노랑 → 초록
