# SC-50: 결제 구역 진입 → 결제 완료 → TRACKING_CHECKOUT

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → (결제 구역 진입, SM 유지) → `TRACKING_CHECKOUT` |
| **트리거** | BoundaryMonitor `on_checkout_enter` → 결제 팝업 → `payment_success` → `enter_tracking_checkout` |
| **우선순위** | HIGH |
| **연관** | UR-40, UR-40a, SR-32, SR-32a |

---

## 사전 조건

- SM = `TRACKING`
- 장바구니에 미결제 항목 존재
- BOUNDARY_CONFIG DB에 `payment_zone` 좌표 입력
- 로봇이 결제 구역(zone 150) 밖에 위치

## 시험 절차

1. **로봇을 결제 구역(zone 150) 안으로 이동**
   - 실 환경: 인형을 결제 구역으로 유도
   - Gazebo/테스트: AMCL pose 직접 시뮬레이션 또는 BoundaryMonitor mock
   ```bash
   # BoundaryMonitor 강제 트리거 테스트:
   ros2 topic pub /robot_54/cmd std_msgs/String \
     '{"data": "{\"cmd\": \"checkout_zone_enter_test\"}"}'
   ```

2. **SM 상태 유지 확인** — `TRACKING` 그대로
   ```bash
   ros2 topic echo /robot_54/status
   # mode: TRACKING
   ```

3. **[앱] 결제 팝업(3-F) 표시 확인** (`checkout_zone_enter` WebSocket 수신)
   - 장바구니 목록 + 합계 표시
   - LCD: 결제 안내 메시지 표시

4. **가상 결제 수행**
   - [앱] 결제 팝업에서 [결제하기] 클릭 → control_service 결제 처리
   - control_service → Pi: `{"cmd": "payment_success"}`
   - Pi: `mark_items_paid()` → 미결제 항목 `is_paid=1` 갱신
   - Pi: `sm.trigger('enter_tracking_checkout')`

5. **SM = `TRACKING_CHECKOUT` 전환 확인**

## 예상 결과

- SM = `TRACKING_CHECKOUT`
- [앱] 결제 팝업 닫힘, 뱃지: "추종 중" → "추종 중 (결제완료)" (청록)
- CART_ITEM `is_paid=1` 갱신 확인
- [앱] 장바구니: 결제 완료 항목 ✓ 표시, [✕] 비활성화
- LED: 초록 유지 (TRACKING과 TRACKING_CHECKOUT 동일)
