# SC-10: 주인 놓침 → 재발견 → 추종 복귀

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → `SEARCHING` → `TRACKING` |
| **트리거** | `enter_searching` → `resume_tracking()` |
| **우선순위** | HIGH |
| **연관** | UR-11, SR-42, SR-22 |

---

## 사전 조건

- SM = `TRACKING`
- `doll_detector` 동작 중 (실 카메라 또는 Mock 설정 변경)

## 시험 절차

1. **인형을 카메라 시야에서 제거** (또는 Mock: `N_MISS_FRAMES` 이상 None 반환하도록 설정)
   - BT 1: 미발견 카운터가 `N_MISS_FRAMES(=30)` 도달
   - 예상: `enter_searching` 트리거

2. **SM 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```
   - 예상: `mode = SEARCHING`

3. **로봇 제자리 회전 확인** (`/cmd_vel` angular.z ≠ 0)
   ```bash
   ros2 topic echo /robot_54/cmd_vel
   ```

4. **인형을 카메라 시야에 다시 제시** (또는 Mock: `get_latest()` 유효값 반환 복구)
   - BT 2: `doll_detector.get_latest() != None` → `sm.resume_tracking()`

5. **SM 전환 확인**

## 예상 결과

- SM = `TRACKING` (이전 상태가 TRACKING이었으므로 TRACKING 복귀)
- [앱] 상단 뱃지: "탐색 중" → "추종 중"
- LCD: "주인을 찾고 있습니다" → "추종 중"
- LED: 주황 → 초록

## 변형 테스트

- `previous_tracking_state = TRACKING_CHECKOUT`인 상태에서 동일 절차 반복
  → 복귀 후 SM = `TRACKING_CHECKOUT`, 뱃지 = "추종 중 (결제완료)"
