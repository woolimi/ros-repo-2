# SC-30: 상품 검색 → 안내 → 도착 → 추종 복귀

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → `GUIDING` → `WAITING` → `TRACKING` |
| **트리거** | `enter_guiding` → `enter_waiting` (도착) → `resume_tracking` ([확인]) |
| **우선순위** | HIGH |
| **연관** | UR-32, UR-33, SR-44, SR-72 |

---

## 사전 조건

- SM = `TRACKING`
- DB ZONE 테이블에 음료(zone_id=6) 등록, waypoint 좌표 입력 완료
- LLM 서버 기동 (또는 Mock: `/query?name=음료` → `{"zone_id": 6, "zone_name": "음료"}`)
- Nav2 기동 및 초기 위치 설정

## 시험 절차

1. **[앱] [🔍 상품 찾기] 클릭 → 검색 패널(3-D) 열기**

2. **[앱] "음료" 텍스트 입력 → [검색] 클릭**
   - customer_web → LLM(채널 D): `GET /query?name=음료`
   - LLM 응답: `{"zone_id": 6, "zone_name": "음료"}`
   - customer_web → control_service(채널 C): `{"cmd": "navigate_to", "zone_id": 6}`

3. **control_service 처리 확인**
   - DB 조회: zone_id=6 waypoint 좌표 취득
   - Pi cmd: `{"cmd": "navigate_to", "zone_id": 6, "x": ..., "y": ..., "theta": ...}`

4. **SM = `GUIDING` 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```

5. **BT 4 동작 확인**
   - Nav2 Goal 전송 로그
   - 로봇 목적지로 이동 (RViz 또는 맵 오버레이 확인)

6. **목적지 도착**
   - BT 4: Nav2 SUCCESS → `arrived` 이벤트 → `enter_waiting`

7. **[앱] 도착 팝업(3-H) 표시 확인**
   - "음료 코너에 도착했습니다."

8. **[앱] [확인] 클릭**
   - customer_web → Pi: `{"cmd": "resume_tracking"}`

## 예상 결과

- SM = `TRACKING`
- [앱] 뱃지: "안내 중" → "대기 중" → "추종 중"
- LCD: "목적지 안내 중" → "대기 중" → "추종 중"
- LED: 노랑 → 파랑 → 초록

## 변형 테스트

- `previous_tracking_state = TRACKING_CHECKOUT` → 도착 후 SM = `TRACKING_CHECKOUT`
