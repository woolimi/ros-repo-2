# SC-21: WAITING 타임아웃 → 빈 카트 귀환

| 항목 | 내용 |
|---|---|
| **상태 전환** | `WAITING` → `RETURNING` → `CHARGING` |
| **트리거** | `enter_returning` (타임아웃 + 빈 카트) → `enter_charging` |
| **우선순위** | MEDIUM |
| **연관** | UR-12, UR-43, SR-34, SR-35 |

---

## 사전 조건

- SM = `WAITING`
- 장바구니 비어있음 (`CART_ITEM` 없거나 모두 `is_paid=1`)
- `WAITING_TIMEOUT = 300`초 (테스트 시 단축: `config.py` 임시 조정 권장)

## 시험 절차

1. **SM = `WAITING` 상태 진입 확인** (SC-11 또는 SC-20에서 이어서 진행)

2. **장바구니 비어있음 확인**
   ```bash
   ros2 topic echo /robot_54/cart
   ```

3. **WAITING_TIMEOUT 대기**
   - SM: `has_unpaid_items()` = False → `enter_returning`

4. **SM = `RETURNING` 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   ```

5. **BT 5 동작 확인**
   - Keepout Filter 활성화 로그 확인
   - `/zone/parking/available` REST 호출 → `zone_id = 140 or 141` 응답
   - Nav2 Goal 전송 확인

6. **충전소 도착 후 SM = `CHARGING` 전환 확인**

## 예상 결과

- SM = `CHARGING`
- `SESSION.is_active` = 0 (세션 종료)
- `ROBOT.active_user_id` = NULL
- `CART_ITEM` 전체 삭제
- [앱] 세션 종료 → `login.html` 리다이렉트
- LED: 보라(RETURNING) → 빨강(CHARGING)
- Keepout Filter 비활성화 (도착 후)
