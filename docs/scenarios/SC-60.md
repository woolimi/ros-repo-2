# SC-60: 정상 귀환 — 빈 카트

| 항목 | 내용 |
|---|---|
| **상태 전환** | `TRACKING` → `RETURNING` → `CHARGING` |
| **트리거** | `enter_returning` (보내주기, 빈 카트) → `enter_charging` |
| **우선순위** | HIGH |
| **연관** | UR-42, UR-43, SR-33, SR-35, SR-44 |

---

## 사전 조건

- SM = `TRACKING`
- 장바구니 비어있음 (`is_paid=0` 항목 없음)

## 시험 절차

1. **[앱] [🚪 쇼핑 종료] 클릭**
   - 장바구니 비어있으므로 확인 다이얼로그 없이 (또는 "장바구니가 비어있습니다" 확인)
   - customer_web → control_service: `{"cmd": "return"}`
   - control_service → Pi: `{"cmd": "mode", "value": "RETURNING"}`
   - Pi: `sm.trigger('enter_returning')`

2. **SM = `RETURNING` 전환 확인**
   ```bash
   ros2 topic echo /robot_54/status
   # mode: RETURNING
   ```

3. **BT 5 동작 확인**
   - Keepout Filter 활성화 로그
   - REST `/zone/parking/available` 호출 → `zone_id` 응답 (140 또는 141)
   - Nav2 Goal 전송 → 로봇 충전소 방향 이동

4. **충전소 슬롯 도착**
   - BT 5: Nav2 SUCCESS → Keepout Filter 비활성화 → `sm.trigger('enter_charging')`

5. **SM = `CHARGING` 전환 확인**

## 예상 결과

- SM = `CHARGING`
- `SESSION.is_active` = 0 (세션 종료)
- `ROBOT.active_user_id` = NULL
- `CART_ITEM` 전체 삭제
- [앱] `staff_resolved`는 없고 `session_end` → `login.html` 리다이렉트
- LED: 초록 → 보라(RETURNING) → 빨강(CHARGING)
- Keepout Filter 비활성화
