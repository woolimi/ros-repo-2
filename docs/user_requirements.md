# 사용자 요구사항 (User Requirements)

> **프로젝트:** 쑈삥끼 (ShopPinkki)
> **팀:** 삥끼랩 | 에드인에듀 자율주행 프로젝트 2팀

---

## UR 테이블

### 진입 / 계정

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-01 | 진입 | **HIGH** | 고객은 쑈삥끼 LCD에 항상 표시되는 QR 코드를 스캔하여 웹앱에 접속할 수 있다. |
| UR-02 | 계정 / 로그인 | **HIGH** | 웹앱 최초 접속 시 로그인을 한다. MVP에서는 하드코딩된 계정으로 동작한다. |
| UR-03 | 계정 / 인형 등록 | **HIGH** | 로그인 직후 주인 인형 등록을 수행한다. 로봇이 IDLE 상태에서 카메라로 인형을 인식하면 등록이 완료되어 추종을 시작한다. |
| UR-04 | 세션 | **MEDIUM** | 세션이 유효한 상태에서 QR 코드를 재접속하면 로그인 및 인형 등록을 건너뛰고 메인화면으로 이동한다. |
| UR-05 | 세션 / 사용 중 차단 | **LOW** | 이미 사용 중인 쑈삥끼의 QR 코드를 다른 사람이 스캔하면 "현재 사용 중입니다" 메시지를 표시하고 접속을 차단한다. |
| UR-06 | 세션 / 중복 사용 차단 | **LOW** | 이미 다른 쑈삥끼를 사용 중인 사용자가 새 쑈삥끼에 로그인을 시도하면 "이미 다른 카트를 사용 중입니다" 메시지를 표시하고 로그인을 차단한다. |

### 추종 / 탐색

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-10 | 추종 | **HIGH** | 인형 등록이 완료되면 쑈삥끼는 주인 인형을 인식하고 마트 안에서 졸졸 따라다니며 쇼핑을 보조한다. |
| UR-11 | 추종 / 재탐색 | **HIGH** | 쑈삥끼가 주인 인형을 놓쳤을 경우, 제자리에서 방향을 전환하며 인형을 재탐색한다. 재탐색에도 찾지 못하면 대기 모드로 전환한다. |
| UR-12 | 대기 | **MEDIUM** | 주인은 웹앱을 통해 쑈삥끼를 대기 모드로 전환하거나, 다시 추종 모드로 복귀시킬 수 있다. (화장실 등 자리 비움 시 활용) 일정 시간 내 복귀하지 않으면 장바구니 상태에 따라 자동으로 LOCKED 또는 RETURNING으로 전환된다. |
| UR-13 | 대기 / 통행 방해 회피 | **LOW** | 대기 중 쑈삥끼가 통행을 방해하는 위치에 있을 경우, 주변 통행자를 감지하여 자동으로 비켜준다. |
| UR-14 | 배터리 / 정지 | **MEDIUM** | 쑈삥끼의 배터리가 일정 수준 이하로 떨어지면 장바구니 상태나 현재 동작에 관계없이 즉시 그 자리에서 정지(HALTED)하고, 앱과 LCD에 배터리 부족 알림을 표시한다. 스태프가 직접 와서 처리할 때까지 대기한다. |

### 앱 / 상태 확인

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-20 | 앱 / 상태 확인 | **HIGH** | 주인은 웹앱에서 마트 맵 위에 모든 쑈삥끼의 실시간 위치와 동작 모드를 확인할 수 있다. 본인의 쑈삥끼는 강조 표시하고, 다른 쑈삥끼는 연하게(투명도 낮춰) 표시하여 시각적으로 구분한다. |
| UR-21 | LCD / 상태 표시 | **HIGH** | 쑈삥끼 LCD는 현재 동작 모드에 따라 화면을 다르게 표시한다. 배터리 잔량은 모든 상태에서 항상 표시한다. (LCD는 터치 조작 불가 — 조작은 웹앱에서만 가능) 상태별 LCD 표시는 아래와 같다. |

**UR-21 상태별 LCD 표시:**

| 상태 | LCD 표시 내용 |
|---|---|
| `CHARGING` | 충전 중 아이콘 + 배터리 잔량 |
| `IDLE` | 웹앱 접속용 QR 코드 + "인형을 카메라에 보여주세요" 안내 |
| `TRACKING` | Pi 카메라 실시간 영상 + 주인 인형 바운딩박스 |
| `TRACKING_CHECKOUT` | Pi 카메라 실시간 영상 + 주인 인형 바운딩박스 (TRACKING과 동일) |
| `GUIDING` | 목적지 안내 중 표시 + 목적지 구역명 |
| `SEARCHING` | "주인을 찾고 있습니다" 안내 |
| `WAITING` | 웹앱 접속용 QR 코드 + "대기 중" 안내 |
| `LOCKED` | "잠금 상태 — 직원을 호출합니다" 안내 |
| `RETURNING` | "충전소로 복귀 중" 안내 (`is_locked_return`이면 잠금 표시 유지) |
| `HALTED` | "배터리 부족 — 직원을 호출합니다" 안내 |

### 쇼핑 / 장바구니

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-30 | 쇼핑 리스트 / 추가 | **HIGH** | 주인은 웹앱에서 "물건 추가" 모드를 선택한 후 쑈삥끼의 카메라에 상품 QR 코드를 보여주어 장바구니에 물건을 추가할 수 있다. |
| UR-31 | 쇼핑 리스트 / 삭제 | **MEDIUM** | 주인은 웹앱의 장바구니 목록에서 항목 옆 삭제 버튼을 눌러 물건을 제거할 수 있다. |
| UR-32 | 물건 찾기 | **MEDIUM** | 주인은 웹앱에서 텍스트 입력 또는 음성-텍스트 변환(STT)을 통해 찾는 물건의 위치를 쑈삥끼에게 질의할 수 있다. |
| UR-33 | 물건 안내 | **HIGH** | 쑈삥끼는 물건이 진열된 위치로 이동하여 주인에게 위치를 안내한 뒤, 자동으로 추종을 재개한다. 도착 시 웹앱 알림으로 주인에게 알린다. |

### 결제 / 귀환

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-40 | 결제 / 구역 진입 | **HIGH** | 쑈삥끼가 결제 구역에 진입해도 TRACKING 상태를 유지한다. 동시에 앱에 결제 여부를 묻는 알람 팝업이 표시되고, LCD에는 결제 화면으로 유도하는 QR 코드가 표시된다. |
| UR-40a | 결제 / 출구 통제 | **HIGH** | 결제가 완료되지 않은 카트는 결제 구역을 지나 출구로 이동할 수 없다. 결제 완료 후에만 TRACKING_CHECKOUT 상태로 전환되어 출구 통과가 허용된다. |
| UR-40b | 결제 / 쇼핑 재개 | **MEDIUM** | TRACKING_CHECKOUT 상태에서 사용자가 다시 결제 구역 안쪽으로 돌아와 쇼핑을 재개하면 TRACKING 상태로 전환된다. 카트는 이미 결제된 물건 목록을 기억하며, 새로 추가된 물건에 대해서만 다음 결제 구역 진입 시 결제를 진행할 수 있다. |
| UR-41 | 결제 / 가상 결제 | **LOW** | 결제는 등록된 카드 정보를 기반으로 가상 결제로 처리한다. 실제 결제 API 연동은 없다. |
| UR-42 | 귀환 | **HIGH** | 주인이 웹앱에서 "보내주기"를 선택하면 쑈삥끼는 충전소로 자동 복귀한다. 장바구니에 미결제 물건이 있으면 LOCKED 상태로 전환 후 귀환하고, 비어있으면 RETURNING 상태로 바로 귀환한다. |
| UR-43 | 귀환 / 세션 종료 | **HIGH** | 정상 귀환(RETURNING) 시 충전소 도착과 함께 세션이 종료되고 인형 등록 데이터가 삭제된다. LOCKED 귀환의 경우 스태프가 처리를 완료한 후 세션이 종료된다. |

### 잠금 / 정지 / 스태프 처리

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-60 | 잠금 / LED 식별 | **HIGH** | LOCKED 상태에서 귀환(RETURNING) 및 충전소 도착(CHARGING) 후에도 LED는 잠금 신호를 계속 표시한다. 스태프가 처리를 완료하기 전까지 LED 잠금 신호가 유지되어 직원이 해당 카트를 즉시 식별할 수 있다. |
| UR-61 | 정지 / 스태프 처리 | **HIGH** | HALTED 상태의 카트는 그 자리에서 정지 유지하며 LED와 LCD로 배터리 부족 신호를 표시한다. 스태프가 직접 충전 스테이션으로 이동시킨 후 관제 대시보드에서 수동으로 초기화한다. |
| UR-62 | 잠금 / 스태프 처리 | **HIGH** | LOCKED 카트가 충전 스테이션에 도착하면 LED 잠금 신호로 스태프를 호출한다. 스태프는 관제 대시보드에서 해당 카트의 잠금을 해제(staff_resolved)하여 정상 충전 상태로 전환할 수 있다. |

### 관제

| UR ID | 카테고리 | Priority | Description |
|---|---|---|---|
| UR-50 | 중앙 관제 | **HIGH** | 직원은 중앙 관제 대시보드에서 마트 맵 위에 모든 쑈삥끼의 실시간 위치와 현재 동작 모드를 한눈에 확인할 수 있다. |
| UR-51 | 중앙 관제 / 이벤트 | **MEDIUM** | 배터리 부족(HALTED), 잠금(LOCKED) 등 이벤트가 발생하면 중앙 관제 대시보드에 알람이 표시되고 로그로 기록된다. |
| UR-52 | 중앙 관제 / 강제 종료 | **MEDIUM** | 직원은 관제 대시보드에서 임의의 쑈삥끼에 강제 종료 명령을 내려 즉시 CHARGING 상태로 초기화할 수 있다. |
| UR-53 | 중앙 관제 / 스태프 처리 | **HIGH** | 직원은 관제 대시보드에서 LOCKED 또는 HALTED 상태의 카트를 선택하여 잠금 해제(staff_resolved) 처리를 수행할 수 있다. 처리 완료 시 해당 카트는 정상 CHARGING 상태로 전환되고 세션이 종료된다. |
